Free vs Premium Modules

This matrix describes what users can do in each mode.

Mode definitions:

  • Premium Mode: Connected socket and Companion Bridge supports update/roll actions.
  • Free Mode: Connected socket but Companion Bridge rejects interactions from the app.
  • Offline Mode: No active socket connection.

Capability Matrix

Feature Premium Online Free Mode Offline Mode
Connect, login, select actor Yes Yes No
Load local cached actor Yes Yes Yes
Browse actor sheet data Yes Yes Yes (cached)
Search and filter rows Yes Yes Yes
Quick tabs pin/unpin Yes Yes Yes
Journal notes Yes Yes Yes
Toggle favorites Synced to Foundry Local-only Local-only
Update actor values (HP, counters, toggles) Synced to Foundry Local-only Local-only
Update item values (spell prepared, item flags) Synced to Foundry Local-only Local-only
Delete active effects Synced to Foundry Local-only Local-only
Cast remote spell/item actions (attack/save/use item) Yes Blocked Blocked
Rest actions (short/long) Yes Blocked Blocked

What Users See In Restricted Modes

When an action is restricted, users may see local-mode feedback such as:

  • Saved locally because server updates are unavailable.
  • Action requires premium tier.
  • Action requires active connection.

Practical interpretation:

  • Sheet edits often continue locally.
  • Server-side game actions (like remote rolls and rest execution) require premium online capability.

Usage Recommendations

  • Use Free version to test the app's features.
  • Use Premium online for full gameplay parity with Foundry actions.
  • Use Offline as continuity mode for tracking and note-taking when full sync is unavailable.
