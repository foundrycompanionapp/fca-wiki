Role-Based And Session Playbook

Use this page for practical session flow, based on player role and game phase.

Host Requirement (Read First)

  • A GM user must be logged into Foundry in a browser while playing.
  • The Companion Bridge module runs in that browser session.
  • If no GM-hosted browser session is active, live sheet loading and bridge interactions are unavailable.

Player vs GM Quick Expectations

Player

  • Primary goal: run your own character from phone/tablet.
  • Typical actions: check stats, cast spells, track HP/resources, manage favorites, write journal notes.
  • In restricted modes, continue with local tracking and reconnect later.

GM (as bridge host)

  • Keep Foundry world open in a browser during play.
  • Ensure the Companion Bridge module is enabled and world is active.
  • Help players reconnect if bridge-host browser session drops.

Session Workflow By Phase

1. Before Session Starts

  1. Confirm a GM browser session is active in Foundry.
  2. Open app and connect to server URL.
  3. Select user and character.
  4. Pin Quick Tabs for your role:
    • Combat-heavy: Core Stats, Spellbook, Inventory, Favorites.
    • Exploration-heavy: Core Stats, Skills, Inventory, Journal.
  5. Do a quick health check:
    • Cast one cantrip or toggle one slot.
    • Toggle one favorite.
    • Open one container.

Expected Result:

  • All core interactions work in your current mode.

2. During Combat

Recommended loop:

  1. Use Core Stats for HP and temporary HP adjustments.
  2. Use Spellbook for cast/slot tracking and preparation toggles.
  3. Use Favorites tab for fastest access to repeated actions.

Tips:

  • If an action is blocked (free/offline), continue with local updates and communicate outcomes at table.
  • Use Journal for quick combat notes (targets, concentration reminders, turn plans).

3. During Exploration And RP

  1. Use Skills and Character tabs for checks and references.
  2. Capture clues and plans in Journal entries.
  3. Keep Favorites curated for recurring utility items/spells.

4. After Encounters

  1. Review HP/temp HP state.
  2. Apply short rest or long rest if available in current mode.
  3. Validate prepared spells and slot circles.
  4. Update journal summary for continuity.

5. End Of Session

  1. Confirm your latest state is synced with the server.
  2. Leave app normally; session restore will attempt to recover on next open.

Fast Recovery Checklist

Use this when things break mid-session:

  1. Verify GM-host browser session is still active.
  2. Reconnect server and reopen actor.
  3. If still unavailable, load local cached actor.
  4. Continue tracking locally until online connection is restored.
  5. If nothing works, feel free to contact us by sending an email or direct message on Patreon.
