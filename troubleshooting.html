Troubleshooting Common Issues

Having issues with the Foundry Companion App? This guide provides solutions to common problems and troubleshooting steps to help you get back on track.

I cannot connect to the server

Checks:

  1. Verify the server URL is correct and reachable.
  2. Confirm Foundry is running and the world is available.
  3. Retry from login step 1 using Change Server.

I can connect but actions are not updating in the Foundry server

Possible cause:

  • You are in Free bridge mode or Offline mode.

What to do:

  1. Check whether the app shows local mode badge.
  2. Reconnect to server and ensure premium-capable bridge is active.
  3. If still restricted, continue in local mode and sync later by reconnecting.

Cast button shows blocked behavior

Possible cause:

  • Remote roll actions are unavailable in current mode.

What to do:

  1. Reconnect and retry in premium online mode.
  2. If unavailable, track spell effects manually and continue using local sheet updates..

Rest action does not execute on server

Possible cause:

  • Rest is blocked in current mode.

What to do:

  1. Confirm connection and bridge tier.
  2. Retry when premium online mode is available.

Favorites changed but not visible where expected

What to do:

  1. Reload actor view or re-open Favorites tab.
  2. Ensure you are still on the same actor.
  3. If local mode, remember changes are local-first until full sync mode is available.

Local actor does not appear in Select Local Actor

What to do:

  1. Load an actor from server at least once to create cache.
  2. Reopen app or return to local actor section.
  3. If needed, clear and rebuild cache by selecting actor again from server flow.

Journal entries seem missing

Checks:

  1. Confirm you are on the same actor; journal is actor-specific.
  2. Verify you did not clear cache for that actor.
  3. Return to Journal tab after actor reload.
